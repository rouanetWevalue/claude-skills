# Analyse & Synthèse

Objectif : comprendre le contenu d'un document, en dégager les points clés, les tensions, les risques ou répondre à des questions de raisonnement.

---

## Protocole général

1. **Lire en entier** — ne pas analyser en diagonale
2. **Identifier le type de document** : spec technique, rapport, contrat, note interne, article...
3. **Dégager la structure** : quels sont les blocs logiques ?
4. **Répondre à la demande** en s'appuyant sur des citations courtes et précises
5. **Distinguer clairement** : fait dans le document / inférence / opinion

---

## Résumé

Structure recommandée pour un résumé :

```
**En une phrase** : [l'essentiel du document]

**Points clés** :
- [Point 1]
- [Point 2]
- [Point 3]

**Ce qui manque / points d'attention** : [lacunes ou ambiguïtés notables]
```

Longueur cible : ~10-15% de la longueur originale sauf instruction contraire.

---

## Analyse de spec / document technique

Points à couvrir systématiquement :

- **Objectif déclaré** : ce que le document dit vouloir accomplir
- **Contraintes identifiées** : techniques, temporelles, budgétaires
- **Hypothèses implicites** : ce qui est supposé vrai sans être dit
- **Risques ou incohérences** : contradictions internes, zones grises
- **Questions ouvertes** : ce qui n'est pas résolu dans le document

---

## Q&A sur document

Format de réponse :

```
**Réponse** : [réponse directe]

**Basé sur** : "[citation courte]" (§X.X ou p.N)

**Nuance** : [si la réponse n'est pas tranchée dans le document]
```

Si la réponse n'est pas dans le document :
> "Cette information ne figure pas dans le document fourni."

Ne jamais compléter avec des connaissances générales sans le signaler explicitement.

---

## Analyse de cohérence

Identifier :
- Contradictions entre sections
- Affirmations non étayées
- Dépendances non résolues (référence à un document ou système non fourni)
- Termes utilisés de manière inconsistante
